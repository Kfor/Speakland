# Speakland - 沉浸式语言学习 APP

## Background

传统语言学习 APP 存在以下痛点：
- 内容枯燥，难以坚持
- 缺乏真实语境，学了不会用
- 反馈滞后，错误得不到及时纠正

Speakland 通过 RPG/Galgame 式的沉浸故事体验，让用户在「玩」中学习语言。用户作为故事主角参与剧情，与 AI 角色互动，在真实语境中自然习得语言。

## Target Users & Scenarios

**核心用户**
- 英语/西班牙语学习者（初中级水平）
- 对传统学习方式感到枯燥的人群
- 希望提升口语和听力的用户

**使用场景**
- 碎片时间娱乐式学习（通勤、睡前）
- 沉浸式口语练习
- 情境化词汇积累

## User Flow

### Onboarding 流程

1. **吉祥物登场**
   - 猫咪精灵（手持书本）出现在界面一侧
   - 全程陪伴用户完成问卷

2. **问卷调查**
   - 学习语种（英语/西班牙语）
   - 学习目的（商务/考试/交流/旅游等）
   - 难度自评
   - 学习类型偏好（口语/单词/听力等）
   - 学习痛点（记不住单词/无聊/无法坚持等）

3. **双语标签**
   - 选择语种后，界面中非基础词汇显示目标语言翻译标注

4. **功能亮点展示**
   - 根据用户痛点动态展示对应功能：
     - 「难以坚持」→ 「NPC 每日发送复习单词」
     - 「无聊」→ 「有趣剧情 + 深度代入感」

5. **剧本方向选择**
   - 职场大女主
   - 和明星谈恋爱
   - 穿越到中世纪
   - 其他场景...

6. **输入游戏昵称**

7. **新手引导世界**
   - 吉祥物带领用户进入引导场景
   - 介绍核心 UI 和功能（词汇交互、口音纠正等）

8. **付费墙**
   - 引导结束后触发
   - 完成购买或关闭后进入主页面

### 主页面 (3 Tabs)

**Tab 1: 故事 (Home)**
- 顶部搜索框
- 世界场景卡片（含封面图、难度标注、简介）
- 推荐世界

**Tab 2: 角色**
- 已解锁的 NPC 角色列表
- 吉祥物为默认角色
- 点击角色可进行一对一对话

**Tab 3: 单词本/设置**
- 生词列表
- 收藏句子
- 口音建议记录
- 顶部登录按钮（登录后同步数据）

### 核心学习场景

**界面布局**
- 背景：场景图片
- 上方：两个角色立绘（静态 + 表情切换）
- 中部：实时字幕 + 语音播放
- 底部：聊天输入框（语音/文字）

**学习功能**

1. **词汇交互**
   - 长按字幕中的单词
   - 弹出：语境翻译 + 发音按钮 + 加入生词本
   - 默认长按即添加，可在弹窗取消

2. **全文翻译**
   - 右上角翻译按钮
   - 点击后显示设备语言翻译

3. **顶部提醒系统**（学习反馈区）
   - 鼓励型：「说得非常地道」+ 小标识
   - 单词改正型：更好的用词/语法建议
   - 口音提醒：音标提示（MVP 后续迭代）

- **移动端**：Expo (React Native) - iOS + Android
- **后端/数据库**：Supabase
- **登录**：Google Sign-In, Apple Sign-In
- **支付**：RevenueCat
- **语音识别**：平台原生或第三方 STT 服务
- **AI 对话**：LLM API（具体方案待定）
- **Web 官网**：独立站点

### 内容需求

**MVP 3 个短故事**
- 类型覆盖：职场/生活/冒险（或根据 Onboarding 热门选项确定）
- 每个故事包含：剧情大纲 + 角色人设 + 分支节点

**吉祥物**
- 形象：猫咪精灵，手持书本
- 用途：Onboarding 陪伴、新手引导、默认对话角色

## Design / Constraints

**交互约束**
- 下方为用户沉浸体验区，不放教学提醒
- 上方为学习反馈区（提醒、纠正）
- 长按默认添加生词，降低操作成本

**技术约束**
- MVP 语音仅做转文字，不做口音检测
- 角色使用静态立绘 + 表情切换，不做 Live2D
- AI 对话需保持角色一致性和剧情连贯性

**内容约束**
- 官方提供剧情大纲和人设
- AI 在框架内实时生成对话
- 用户选择可影响分支走向

## Out of Scope (MVP)

- 口音检测与详细纠正
- Live2D 角色动画
- 用户自定义故事/角色
- 社交功能（好友、排行榜）
- 离线模式
- 英语/西班牙语以外的语种
